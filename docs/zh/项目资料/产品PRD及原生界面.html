1.1 产品愿景

CardScope 致力于成为全球球星卡收藏者的首选 AI 助手，通过拍照即刻识别球星卡的球员、年份、系列、编号，并提供实时市场估值，让每一位收藏者都能"一拍即知价值"。

1.2 产品定位

一句话定位：拍照识别球星卡，秒知球员信息与市场价值。

Slogan："Scan. Identify. Value. — Your AI Card Expert."

1.3 目标市场

  • 核心市场：美国（球星卡市场最大，占全球 70%+）
  • 扩展市场：欧洲（足球卡）、日本（棒球/动漫卡）、澳洲
  • 市场规模：全球体育卡牌市场约 $300 亿（2024），年增长 10%+

👤 2. 目标用户画像

Persona A — 新手收藏者 "Jake"

  • 年龄：22-35岁
  • 背景：刚入门球星卡收藏，在 eBay/Card Show 淘卡
  • 痛点：不了解卡面信息、不知道卡值多少钱、怕买到假卡
  • 需求：快速识别 + 估价，避免被坑
  • 使用频率：每周 3-5 次

Persona B — 资深藏家 "Mike"

  • 年龄：35-55岁
  • 背景：收藏 10 年+，拥有上千张卡，部分已送评（PSA/BGS）
  • 痛点：需要批量管理收藏、追踪市值变化、做投资决策
  • 需求：收藏管理 + 市值追踪 + Portfolio 分析
  • 使用频率：每天 1-3 次

Persona C — 卡贩/Reseller "Sarah"

  • 年龄：25-40岁
  • 背景：在 eBay/Mercari/COMC 上买卖球星卡
  • 痛点：需要快速查卡信息、实时比价、高效上架
  • 需求：批量扫描 + 一键获取 comp 价格
  • 使用频率：每天 10+ 次（高频用户）

3.2 核心流程：AI 识别

用户点击"扫描"按钮 ↓ 打开相机（或选择相册图片） ↓ 对准球星卡拍照 ↓ AI 图像分析（1-3 秒） ├── 端侧预处理：裁剪、校正、去光斑 ├── 云端识别：匹配球员、系列、年份、编号、变体（Parallel） └── 估值计算：拉取近期 eBay sold comps ↓ 展示识别结果页 ├── 球员信息（姓名、球队、位置、球衣号） ├── 卡面信息（品牌、系列、年份、卡号、Parallel 类型） ├── 市场估值（Raw / PSA 9 / PSA 10 分别估价） ├── 近期成交记录 └── 操作：收藏 / 分享 / 查看详情 ↓ 用户选择"添加到收藏" ↓ 自动归类到对应球员/系列文件夹

3.3 AI 识别详细规格

卡号识别OCR 读取卡背编号>90%
变体/Parallel 识别Silver / Gold / Green / Holo 等>85%
品相预估Centering / Corners / Edges / Surface±1 grade

3.4 支持的卡牌类型（MVP）

  • 运动类型：NBA 篮球（优先）、NFL 橄榄球、MLB 棒球、足球（Soccer）
  • 品牌覆盖：Panini Prizm、Panini Select、Panini Mosaic、Topps Chrome、Topps Heritage、Upper Deck、Donruss、Hoops
  • 年份范围：2010-2025（MVP），后续扩展到 1980+
  • 预估覆盖：~500,000+ 张不同卡面

6.1 整体架构

┌─────── Client (iOS) ───────┐ │ SwiftUI + Swift 5.9+ │ │ Core ML (端侧预处理) │ │ AVFoundation (相机) │ │ Vision Framework (OCR) │ │ Charts (价格走势) │ │ StoreKit 2 (订阅) │ └──────────┬─────────────────┘ │ HTTPS / REST API ┌──────────▼─────────────────┐ │ Backend (Cloud) │ │ ├── API Gateway │ │ ├── Auth Service (Firebase)│ │ ├── Card Recognition AI │ │ │ ├── Image Preprocessing│ │ │ ├── CNN Classifier │ │ │ ├── Vector Search DB │ │ │ └── OCR Pipeline │ │ ├── Pricing Engine │ │ │ ├── eBay API Scraper │ │ │ ├── Price Calculator │ │ │ └── Cache Layer │ │ ├── User Data (Firestore) │ │ └── CDN (Card Images) │ └─────────────────────────────┘

6.2 AI 模型方案

模块方案说明
图像预处理Core ML + Vision端侧裁剪、校正、增强，减少云端传输
卡面匹配CLIP / DINOv2 + Vector DB提取视觉特征向量 → Pinecone/Milvus 检索最相似卡
文字识别Vision OCR + GPT-4V读取卡号、球员名、品牌文字
Parallel 识别Fine-tuned ViT区分 Silver/Gold/Holo 等变体（关键差异化）
品相评估Custom CNN分析 centering/corners/surface 状况

6.3 价格数据来源

  • eBay Sold Listings API: 最近 90 天成交数据（主要数据源）
  • COMC (Check Out My Cards): 在线卡店定价参考
  • PSA Cert Verification: 验证 PSA 编号获取评级信息
  • Card Ladder / Market Movers: 第三方价格追踪
  • 自建爬虫 + 清洗管道： 定时抓取，标准化存储

🔍 8. 竞品分析

竞品核心功能优势劣势
Card Scanner (CardBase)扫描+估值数据库大，估值较准UI 老旧，体验差
Collectr收藏管理+估值收藏管理功能强识别能力弱，需手动搜索
TCGPlayer扫描+交易最大交易平台仅 TCG 卡，非体育卡
PSA/BGS App验证评级权威评级数据功能单一
eBay买卖+比价最大交易量非专业识别工具

CardScope 差异化定位

  • 🎯 AI 优先：拍照即识别，无需手动搜索（竞品多为手动输入）
  • 🎯 Parallel 识别：区分银、金、Holo 变体（竞品做不好的领域）
  • 🎯 现代 UI/UX：对标 PictureThis 级别的用户体验
  • 🎯 品相 AI 预评：发送评级前先 AI 预估，降低送评风险
  • 🎯 Portfolio 追踪：像炒股一样管理卡片投资

⚠️ 10. 风险与应对

风险影响应对策略
Parallel 识别准确率不达标🔴 高初期人工标注 + 用户反馈修正闭环
eBay API 限制/变更🔴 高多数据源备份 + 自建爬虫缓存
数据库覆盖不全（老卡/冷门卡）🟡 中MVP 聚焦 2015+热门系列，逐步扩展
同类竞品抄袭/跟进🟡 中快速迭代 + 数据飞轮 + 品牌建设
Panini/Topps 版权投诉🟡 中合理使用原则 + 用户上传内容 + 法律咨询
云端 AI 推理成本高🟡 中端侧预处理 + 模型量化 + 缓存策略
