## 一、数据需求总览

构建球星卡识别 App 需要 **四层数据**：

```
┌──────────────────────────────────────────────┐
│  第1层：卡面数据库（Card Database）              │
│  → 品牌、系列、年份、卡号、Parallel、球员对应关系  │
├──────────────────────────────────────────────┤
│  第2层：价格数据（Pricing Data）                 │
│  → 实时/历史成交价、按品相分级估值               │
├──────────────────────────────────────────────┤
│  第3层：球员信息（Player Data）                  │
│  → 姓名、球队、位置、赛季数据、头像              │
├──────────────────────────────────────────────┤
│  第4层：图像数据（Image Data）                   │
│  → 卡面高清图片，用于 AI 模型训练                │
└──────────────────────────────────────────────┘
```

### 3.1 ✅ eBay Sold Listings API（最重要的价格源）

| 维度 | 详情 |
|------|------|

| **为什么最重要** | eBay 是全球最大的球星卡交易平台，sold comps 是行业定价标准 |

```
GET /buy/browse/v1/item_summary/search
?q=luka+doncic+2018+prizm+silver+280
&filter=buyingOptions:{FIXED_PRICE|AUCTION},
        conditions:{NEW},
        deliveryCountry:US
&sort=newlyListed
&limit=50

# 关键参数
```

## 五、第4层：图像数据（AI 训练用）

### 5.1 图像数据获取策略

```
图像数据来源优先级：

1️⃣ eBay Listings 图片（最大量、最真实）
   └── 通过 eBay API 获取 sold listings 图片 URL
   └── 优点：真实拍摄场景、各种角度/光线
   └── 数量：每系列可获取 10,000+ 张图

2️⃣ COMC 卡片图片（高质量、标准化）
   └── 每张卡片都有标准化扫描图
   └── 优点：背景统一、清晰度高
   └── 适合：作为"标准参考图"

3️⃣ TCGPlayer 产品图
   └── API 返回中包含图片 URL
   └── 覆盖面广

4️⃣ 用户 UGC（上线后收集）
   └── 用户拍照 + 确认识别结果 = 免费标注数据
   └── 飞轮效应：用户越多 → 数据越多 → 模型越准 → 用户越多

5️⃣ 自建拍摄（最高质量但成本高）
   └── 购买/借用卡片 → 多角度、多光线拍摄
   └── 适合：核心热门卡片的高质量训练样本
```

### 5.2 图像爬取实操方案

```python
# eBay 图片批量获取（伪代码）

import requests

# Step 1: 搜索已售卡片
def search_sold_cards(player, set_name, year):
    url = "https://api.ebay.com/buy/browse/v1/item_summary/search"
    params = {
        "q": f"{year} {player} {set_name}",
        "filter": "buyingOptions:{AUCTION|FIXED_PRICE}",
        "sort": "newlyListed",
        "limit": 200
    }
    headers = {"Authorization": f"Bearer {ACCESS_TOKEN}"}
    resp = requests.get(url, params=params, headers=headers)
    return resp.json()["itemSummaries"]

# Step 2: 提取图片 URL
def extract_images(items):
    images = []
    for item in items:
        if "image" in item:
            images.append({
                "url": item["image"]["imageUrl"],
                "title": item["title"],
                "price": item.get("price", {}).get("value"),
                "item_id": item["itemId"]
            })
    return images

# Step 3: 下载并按分类存储
```

### 6.2 数据库 Schema 设计

```sql
-- 核心数据表结构

-- 球员表
CREATE TABLE players (
    id UUID PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    sport ENUM('basketball','football','baseball','soccer'),
    team VARCHAR(255),
    position VARCHAR(100),
    jersey_number INT,
    rookie_year INT,
    headshot_url TEXT,
    bio TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 卡片产品表（核心）
CREATE TABLE cards (
    id UUID PRIMARY KEY,
    player_id UUID REFERENCES players(id),
    brand VARCHAR(100),        -- Panini, Topps, Upper Deck
    set_name VARCHAR(255),     -- Prizm, Chrome, Select
    year INT,
    card_number VARCHAR(50),   -- #280
    parallel VARCHAR(100),     -- Silver, Gold, Green, Base
    subset VARCHAR(100),       -- Rookie, Insert, Auto
    is_rookie BOOLEAN DEFAULT FALSE,
    is_auto BOOLEAN DEFAULT FALSE,
    is_mem BOOLEAN DEFAULT FALSE,  -- Memorabilia/Relic
    serial_numbered INT,       -- /25, /99, etc.
    image_url TEXT,
    tcgplayer_id VARCHAR(100),
    ebay_search_query TEXT,    -- 预构建的搜索词
    created_at TIMESTAMP DEFAULT NOW()
);

-- 价格数据表
CREATE TABLE prices (
    id UUID PRIMARY KEY,
    card_id UUID REFERENCES cards(id),
    condition VARCHAR(50),     -- Raw, PSA 8, PSA 9, PSA 10, BGS 9.5
    sale_price DECIMAL(10,2),
    sale_date DATE,
    source VARCHAR(50),        -- ebay, comc, tcgplayer
    source_url TEXT,
    listing_title TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 价格聚合表（定时计算）
CREATE TABLE price_summary (
    card_id UUID REFERENCES cards(id),
    condition VARCHAR(50),
    avg_price_30d DECIMAL(10,2),
    median_price_30d DECIMAL(10,2),
    min_price_30d DECIMAL(10,2),
    max_price_30d DECIMAL(10,2),
    total_sales_30d INT,
    price_trend_pct DECIMAL(5,2),  -- 30天涨跌幅
    last_updated TIMESTAMP,
    PRIMARY KEY (card_id, condition)
);

-- AI 训练图片表
CREATE TABLE card_images (
    id UUID PRIMARY KEY,
    card_id UUID REFERENCES cards(id),
    image_url TEXT,
    source VARCHAR(50),         -- ebay, comc, user
    is_front BOOLEAN DEFAULT TRUE,
    quality_score FLOAT,        -- AI 评估图片质量
    is_verified BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_cards_player ON cards(player_id);
CREATE INDEX idx_cards_search ON cards(brand, set_name, year);
CREATE INDEX idx_prices_card ON prices(card_id, sale_date DESC);
CREATE INDEX idx_prices_condition ON prices(card_id, condition);
```

## 八、法律与合规注意事项

### 8.1 数据使用合规

| 数据源 | 合规风险 | 建议 |
|--------|---------|------|
| **eBay API** | 🟢 低 — 使用官方 API，遵守 TOS | 标注"数据来源 eBay"，不缓存超过 TOS 允许时长 |
| **TCGPlayer API** | 🟢 低 — 官方 API | 遵守 API TOS |
| **eBay 爬虫** | 🟡 中 — 超出 API 范围的爬取 | 控制频率，使用 API 为主 |
| **COMC 爬虫** | 🟡 中 — 无公开 API | robots.txt 遵守，考虑合作 |
| **卡面图片** | 🟡 中 — 版权属于 Panini/Topps | 用户上传 = UGC（避免版权），参考图用小尺寸缩略图 |
| **Beckett 数据** | 🔴 高 — 付费数据，严格版权 | 必须商业授权 |

### 8.2 关键建议

```
合规安全策略：

1. ✅ 优先使用官方 API（eBay, TCGPlayer, PriceCharting）
2. ✅ 用户拍照上传 = UGC（合法）
3. ✅ 标注数据来源（"Prices based on recent eBay sales"）
4. ✅ 不存储高清卡面图片（仅存缩略图+链接）
5. ✅ 图片仅用于 AI 训练（fair use / transformative use）
6. ⚠️ 不直接复制 Beckett Book Value
7. ⚠️ 不抓取超过 robots.txt 允许的内容
8. ⚠️ 准备好应对 DMCA takedown 的流程
```
